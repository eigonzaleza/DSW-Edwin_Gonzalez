Conceptos

GIT

Git, es una herramienta que tiene como propósito de controlar las versiones de un proyecto, para el desarrollo de aplicaciones "VCS" traducido al español, aplicación de control de versiones con licenciamiento GNU/GLP V2, desarrollada por el creador de LUNIX el señor LINUS TORVALDS y lanzada en el año 2005.

Git tiene funcionalidades que ayudan al mantenimiento y control de versiones para proyectos indivuales como de proyectos grupales, donde podemos guardar cambios y hacer seguimientos de manera cronológica a nuestros proyectos este es uno de los propósitos principales. También en GIT podemos ramificar como otros los denomina flujo de trabajo, y esto que significa que podemos trabajar en equipo como se mencionó hace un momento, donde GIT va tomando mayor relevancia ya que GIT permite almacenar los registros, ordenarlos y fusionarlos los archivos desde uno hasta "x" archivos. Para un proyecto donde se utilice un archivo la importancia de la aplicación no es significativa, pero cuando los proyectos contegan de varios archivos guardar la versiones e ir a una versión donde el proyecto estaba funcionando bien y después de alguna modiciación o de una actualización la aplicación no funcione seria al algo complejo y a un más cuando se trabaja de menera colaborativa.
